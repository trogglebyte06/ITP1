GLOSSARY OF TERMS

AMBIGUITY: A phenomenon in spoken language allowing a speaker to be understood by fellow humans even when not saying everything perfectly. Ambiguity is not a luxury when communicating to a computer. Computers will only carry out instructions exactly as written. Be careful of typos, missing structural elements, etc. For this reason, computers are often lovingly said to be "stupid".

ATTRIBUTE: A parameter required by some tags which provides additional information to a browser. This information is used by the browser to correctly render tag content in the html document.

BLOCK: A quality of certain tags. Tags are said to be block tags when they create an invisible box around content with height and width. These invisible boxes can be manipulated in various ways to stylize a webpage. We will learn more about this in subsequent lessons. See also: INLINE

BROWSER: A tool used to communicate with servers to view hosted files.

DOCUMENT STRUCTURE: Document Structure refers to additional coding needed to make a functional webpage. Now that we use html4, building the frame of an html document is much simpler. Document Structure should also bring to mind the concept of indentation. When coding, lines of code are indented to make it easier for colleagues to read and interpret. Indentation can also make it easier for a programmer to identify errors in their own code.

DOCUMENTATION: Documentation exists to assist developers when writing code. It is easily found online through a google search. Thanks to documentation it isn't necessary to memorize every html tag. Even professional developers refer to documentation and other online resources on a daily basis to get the job done. The most important skill of a programmer is the ability to creatively solve abstract problems. This is not a memorized skill.

HTML: Hyper Text Markup Language is a language used to write documents which appear on the internet. html documents are the most common form of document on the internet.

HTTP: The language that browsers and servers use to communicate and transfer information over the internet.

INLINE: A quality of certain tags. Tags are said to be inline when the content they markup is seen inline with other content around it. See also: BLOCK

INTERNET, THE: The largest computer network in the world. Invented in the early 90s. Composed of 30 Billion html pages.

SERVER: A computer especially designed to host files which it makes available to browsers upon request.

TAG: A tag is an html operator which is used to manipulate text, images, videos, and other file types on an HTML document.

VOID: A tag is said to be void when no content is required between opening and closing tags. Consequently, no </tag> is needed.

WHITESPACE: The blank spaces between words and paragraphs in a webpage. It should be noted that just because a line is started in the text editor does not mean that a new line will appear in the webpage. For whitespace to appear in the webpage certain tags must be added. See the section below on tags for more information about which tags can be used to add whitespace to a document.

Most tags take extra parameters in the opening tag called attributes which provide additional information to the browser. Many tags take multiple attributes. Note that some tags are said to be void and do not use a closing tag and contain no content (even tags that take multiple attributes).

Examples:
The anchor tag for hyperlinks: 
                  <a href = "google.com">google.com</a> = google.com

WHITESPACE

It is important to remember that line breaks in the text editor do not translate to line breaks on the html document. Multiple spaces in the text editor become only one space in the html document. New lines are only displayed in the html document if the proper coding is implemented. When a new line is needed, there are a variety of tags available for use. Which tag you should use in any given instance depends on how you want your document to be structured.

Examples:
The line break tag, an inline tag. This tag is also void, note that there is no closing tag needed. 
                  <br>
The paragraph tag, creates a different kind of line break involving a block. 
                  <p>content</p>

More on block and inline in the next section.

BLOCK VS INLINE

INLINE tags happen "inline" with the other stuff around it in the html document.

Examples:

The image and line break tags (both seen above) are great examples of inline tags.

BLOCK tags put an invisible box around the content it contains. This box has height and width and can be manipulated in various ways.

Example:
The paragraph tag seen above is a great example of a block tag. See it explicitly in action below. 
                    
<p>here's one block</p>

                    
<p>here's another</p>

There will be more to come on what you can do with inline and block tags in future lessons.

DOCUMENT STRUCTURE

Certain tags are needed to build the frame around an html document. The frame won't change much in this course. Suffice it to say that these tags are needed for html to work correctly.

Example: 
                  <!DOCTYPE html>
                    <html>
                      <head>
                        <title>Title!<title>
                      </head>
                      <body>
                        <b>Content!</b>
                      </body>
                    <html>

Note that indentation is crucial to legible code. See above that the html element above has two child elements, head and body, which are indented the same amount. This indentation rule should apply to all parts of your code. Readable code makes it easier for colleagues to read your code and understand its function. Readable code makes it easier for you, the coder, to go back and troubleshoot problems that come up. Indentation is considered a best practice in the world of computer science generally speaking.
